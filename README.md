# ping
pong the rpg
Welcome to pong but now you are faced with 5 levels each with more health, enemy speed, and ball speed.

Original IDEAS
_enemy ability ideas(skills enemies use)_
- dissapearing ball( a second or two)
- phantom split(one fake splits off from the main one to fake out the player)
- time stop(ball will freeze for a moment to throw the player off)
- reverse pong(ball will change color and the player will need to avoid instead of hitting it)

_Paddle Ideas_
- momentum affects speed and direction of the ball
- ability to change size of paddle for a limited amount of time

_Enviornment ideas_
- different enemies have different room layouts(round, square, ect)
- the different rooms will also have obstacles that appear and or are static

Justin Moos and I participated in the Crimson Code Hackathon. During the hackathon, we were given 24 hours to develop a program that would then be judged in areas of complexity, originality, and adherence to the theme. For this year, the theme was “Re-inventing the wheel”, which meant that our program should be a unique or improved version of something that already exists. 

When we were deliberating ideas, I brought up re-inventing Pong (as it was the first video game), and Justin liked the idea, so that is what we did. I originally wanted to make a RPG-Style Pong, with leveling up, scaling difficulty, and special abilities that could affect the ball or the opponent’s paddle. Ultimately what we were able to complete in the 24 hour timeframe was a fully functional Pong with scaling difficulty, starting out easy, and becoming impossible by the end. 

For our project, we used the Unity Game Engine, coding in C#. I was the one who created the github, while Justin was the one who created the unity project. The main thing I helped with that actually made it into the project was setting up a reaction time/speed change system for the enemy paddle, which would make the digital opponent react faster or more sluggishly depending on the level. I also aided in bugfixing, and worked on spriting (unfortunately these sprites did not end up getting used). 

Overall I wish we could have implemented more of our original ideas. We were having issues with unity version control, as I was unable to pull up the newest committed builds on my computer, so my ability to contribute was limited at certain points. Justin and I have discussed the possibility of continuing this project in the future, but if we did so, it would be after the development of our semester project.
