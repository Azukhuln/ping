# ping
pong the rpg
Welcome to pong but now you are faced with 5 levels each with more health, enemy speed, and ball speed




















Original IDEAS
_enemy ability ideas(skills enemies use)_
- dissapearing ball( a second or two)
- phantom split(one fake splits off from the main one to fake out the player)
- time stop(ball will freeze for a moment to throw the player off)
- reverse pong(ball will change color and the player will need to avoid instead of hitting it)

_Paddle Ideas_
- momentum affects speed and direction of the ball
- ability to change size of paddle for a limited amount of time

_Enviornment ideas_
- different enemies have different room layouts(round, square, ect)
- the different rooms will also have obstacles that appear and or are static

Nicholas and I participated in the hackathon 24 hour event with the theme of this years hackathon being re-inventing the wheel. With the time we were given we wanted to create a more unique version of pong.

Nicholas came up with the idea of re-inventing pong for our project and I thought the idea was interesting as the goal was to make a more difficult game of pong. Our idea was to add abilities and other aspects to make the game more dificult as the player progressed through levels but we ran into roadblocks and time constraints with the event itself. What we decided on with the time we had was to get a working main menu and levels, health for player and bot, and increasing health and difficulty of bot as well as the ball speed as the player progressed. 

For contributions I teamed up with nicholas and we worked as a two man team to make the game. We ended up creating the project in unity with C# which neither of us were framiliar with at all.For individual contributions I spend a majority of the time working on coding and implementing everything into unity. While I was working on this Nicholas helped with coding bugs and implementing certain code sections into the program like the reaction speed for the enemy paddle. Besides bugfixing he helped with creating sprites that we were sadly not able to get working in time for the end of the event. 

At the end of it I feel like we did pretty well with the issues we were running into while developing the project. We were having a hard time working on the unity side at the same time whereas we could code together no issue thanks to an extension for visual studio code which allowed real time editing from both of us. Because we couldn't get the project to load on both of our computers at the same time to develop the project I ended up being the main one working on unity with creating the scenes and we both contributed to the coding. After what we created I would like to dive into it again sometime in the future to futher develop it but for now I want to hunker down on the project we have at hand.
